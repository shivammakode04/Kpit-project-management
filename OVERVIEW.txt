================================================================================
FLOWFORGE - COMPLETE IMPLEMENTATION OVERVIEW
================================================================================

PROJECT STRUCTURE:
- Backend: Django 4.2 + DRF + SimpleJWT + APScheduler
- Frontend: React 19 + Vite + TypeScript + Tailwind + Zustand
- Database: SQLite
- API: RESTful with JWT authentication

================================================================================
BACKEND MODELS
================================================================================

1. USER MODELS (apps/users/)
   - CustomUser: username, email, full_name, avatar_url, role, is_active
   - Fields: id, created_at, updated_at

2. PROJECT MODELS (apps/projects/)
   - Project: name, description, owner(FK), status, is_archived, start_date, end_date
   - ProjectMember: project(FK), user(FK), role(admin/member/viewer), status(pending/accepted)
   - Fields: id, joined_at, created_at

3. STORY MODELS (apps/stories/)
   - UserStory: project(FK), title, description, status, priority, story_points, created_by(FK)
   - Fields: id, task_count, created_at, updated_at

4. TASK MODELS (apps/tasks/)
   - Task: story(FK), title, description, status, priority, assigned_to(M2M), due_date, created_by(FK)
   - Task fields: estimated_hours, actual_hours, task_type, labels, is_blocked, created_at, updated_at
   - TaskAttachment: task(FK), file, filename, file_size, content_type, uploaded_by(FK), created_at
   - Comment: task(FK), user(FK), content, created_at
   - ActivityLog: user(FK), project(FK), action, target_type, target_id, created_at

5. NOTIFICATION MODELS (apps/notifications/)
   - Notification: user(FK), type, title, message, is_read, project(FK), related_object_id, created_at
   - Types: task_assigned, task_updated, task_completed, story_assigned, story_updated, project_invite, deadline_reminder, comment_added, project_updated

6. JOB MODELS (apps/jobs/)
   - BackgroundJob: job_type, status, payload, result, retry_count, max_retries, error_message, scheduled_at, executed_at, created_at

BACKEND API ENDPOINTS
================================================================================

AUTH ENDPOINTS (apps/users/):
- POST /api/auth/register/ - Register new user
- POST /api/auth/login/ - Login (returns JWT tokens)
- POST /api/auth/refresh/ - Refresh access token
- GET /api/auth/profile/ - Get current user profile
- PATCH /api/auth/profile/ - Update profile
- POST /api/auth/profile/avatar/ - Upload avatar
- POST /api/auth/profile/password/ - Change password

PROJECT ENDPOINTS (apps/projects/):
- GET /api/projects/ - List user's projects
- POST /api/projects/ - Create project

- PATCH /api/projects/{id}/ - Update project
- DELETE /api/projects/{id}/ - Delete project
- POST /api/projects/{id}/archive/ - Toggle archive
- GET /api/projects/{id}/members/ - List members (supports ?status=accepted)
- POST /api/projects/{id}/members/ - Add member
- DELETE /api/projects/{id}/members/{user_id}/ - Remove member
- PATCH /api/projects/{id}/members/{user_id}/role/ - Change role
- GET /api/projects/{id}/users-directory/ - Get all users (searchable with ?q=)
- POST /api/projects/{id}/invite/ - Invite user (creates pending member + notification)
- POST /api/projects/invites/{member_id}/accept/ - Accept invitation
- GET /api/projects/{id}/activity/ - Activity log
- GET /api/projects/{id}/analytics/ - Project analytics
- GET /api/projects/{id}/stories/ - List stories
- GET /api/projects/{id}/tasks/ - List all tasks in project
- GET /api/projects/{id}/export/?format=csv - Export CSV
- GET /api/projects/{id}/export/?format=pdf - Export PDF

STORY ENDPOINTS (apps/stories/):
- GET /api/projects/{id}/stories/ - List stories
- POST /api/projects/{id}/stories/ - Create story
- GET /api/stories/{id}/ - Get story detail
- PATCH /api/stories/{id}/ - Update story
- DELETE /api/stories/{id}/ - Delete story

TASK ENDPOINTS (apps/tasks/):
- GET /api/stories/{id}/tasks/ - List tasks for story
- POST /api/stories/{id}/tasks/ - Create task
- GET /api/tasks/{id}/ - Get task detail
- PATCH /api/tasks/{id}/ - Update task
- DELETE /api/tasks/{id}/ - Delete task
- PATCH /api/tasks/{id}/status/ - Update status only
- POST /api/tasks/{id}/assign/ - Assign user
- GET /api/tasks/{id}/comments/ - List comments
- POST /api/tasks/{id}/comments/ - Add comment
- GET /api/tasks/{id}/attachments/ - List attachments
- POST /api/tasks/{id}/attachments/ - Upload file
- GET /api/tasks/my/ - My assigned tasks

NOTIFICATION ENDPOINTS (apps/notifications/):
- GET /api/notifications/ - List notifications
- PATCH /api/notifications/{id}/ - Mark as read
- POST /api/notifications/mark-all-read/ - Mark all read
- GET /api/notifications/unread-count/ - Get unread count

BACKGROUND JOB ENDPOINTS (apps/jobs/):
- GET /api/jobs/ - List job history
- POST /api/jobs/trigger/ - Trigger job manually

SEARCH ENDPOINTS (apps/search/):
- GET /api/search/?q=term - Global search

BACKEND FEATURES IMPLEMENTED
================================================================================

1. AUTHENTICATION & AUTHORIZATION
   - JWT token-based authentication (SimpleJWT)
   - Role-based access control (Admin/Member/Viewer)
   - Custom user model with avatar support
   - Password hashing and validation

2. PROJECT MANAGEMENT
   - Create/Read/Update/Delete projects
   - Archive/Unarchive projects
   - Project member management with roles
   - Project analytics and activity logging

3. TEAM INVITATION WORKFLOW
   - Invite users to projects (creates pending member)
   - Accept/Reject invitations
   - Notification system for invites
   - Status tracking (pending/accepted)

4. STORY & TASK MANAGEMENT
   - Hierarchical structure: Project → Story → Task
   - Task assignment to multiple users (M2M)
   - Task status tracking (todo/in_progress/done/blocked/testing)
   - Priority levels (low/medium/high/critical)
   - Due date tracking
   - Story points estimation

5. BACKGROUND JOBS (APScheduler)
   - deadline_reminder: Runs hourly, sends notifications for tasks due in 24 hours
   - notification_cleanup: Runs daily at 3 AM, deletes read notifications older than 30 days
   - Retry logic with exponential backoff (2s → 4s → 8s)
   - Max 3 retries per job
   - Job execution tracking in BackgroundJob model

6. NOTIFICATIONS
   - In-app notification system
   - Multiple notification types
   - Unread count tracking
   - Mark as read functionality
   - Project-specific notifications

7. FILE ATTACHMENTS
   - Upload files to tasks (5MB limit)
   - MIME type validation
   - File metadata tracking (size, type, uploader)

8. COMMENTS
   - Add comments to tasks
   - User and timestamp tracking

9. ACTIVITY LOGGING
   - Track all project actions
   - User, action, target type, and timestamp

10. SECURITY
    - Input validation and sanitization (bleach)
    - SQL injection prevention (Django ORM)
    - CORS configuration
    - Rate limiting (100/hour for anonymous, 1000/hour for authenticated)
    - File upload validation
